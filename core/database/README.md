# core:database

> Encrypted Room database — SQLCipher-backed persistence implementing all domain repository interfaces

## Overview

`core:database` is Shellify's persistence layer. It wraps [Room](https://developer.android.com/training/data-storage/room) with [SQLCipher](https://www.zetetic.net/sqlcipher/) to provide at-rest encryption for all user data. The encryption passphrase is managed by `core:crypto` so it never appears in plaintext anywhere on disk.

- Convention plugins: `shellify.android.library` + `shellify.ksp`
- Namespace: `io.shellify.core.database`
- Implements: `WebAppRepository`, `CategoryRepository` (from `core:domain`)

## Purpose

Centralise all SQLite I/O behind typed DAOs, enforce foreign-key constraints, and provide reactive `Flow`-based queries so the UI layer always reflects the latest persisted state without polling.

## Key Classes / Files

| Class | Role |
|---|---|
| `AppDatabase` | `@Database` root — opens the SQLCipher-encrypted file; receives passphrase from `CryptoManager.databasePassphrase()` |
| `WebAppEntity` | `@Entity(tableName = "web_apps")` with `@PrimaryKey` |
| `CategoryEntity` | `@Entity(tableName = "categories")` with `@PrimaryKey`; `WebAppEntity` holds a `@ForeignKey` reference |
| `WebAppDao` | `@Dao` — `getAll()`, `getAllAsFlow()`, `getById(id)`, `upsert(entity)`, `deleteById(id)` |
| `CategoryDao` | `@Dao` — same pattern as `WebAppDao` |
| `WebAppMapper` | Maps `WebAppEntity` ↔ `WebApp` domain model |
| `CategoryMapper` | Maps `CategoryEntity` ↔ `Category` domain model |
| `IconSourceConverter` | `@TypeConverter` — serialises/deserialises the `IconSource` sealed class to/from JSON |
| `WebAppRepositoryImpl` | Implements `WebAppRepository`; delegates to `WebAppDao` + `WebAppMapper` |
| `CategoryRepositoryImpl` | Implements `CategoryRepository`; delegates to `CategoryDao` + `CategoryMapper` |

Room migration schemas are auto-generated by KSP into `/app/schemas/`.

## Usage

### Querying data (via injected repository)

```kotlin
class MyViewModel @Inject constructor(
    private val getWebApps: GetWebApps   // use case from core:domain
) : ViewModel() {
    val apps = getWebApps().stateIn(viewModelScope, SharingStarted.Lazily, emptyList())
}
```

Features never import `AppDatabase` or DAOs directly — always go through use cases.

### Adding a new table

1. Create a new `@Entity` data class in `entity/`.
2. Create a `@Dao` interface with the required queries.
3. Add the entity to `AppDatabase.entities` and the DAO accessor function.
4. Create a `Migration(from, to)` object and register it via `AppDatabase.addMigrations(myMigration)`.
5. Add the domain model, repository interface, and use cases in `core:domain`.
6. Implement the repository interface here and add a Hilt `@Binds` binding.

### Adding a Room migration

```kotlin
val MIGRATION_2_3 = object : Migration(2, 3) {
    override fun migrate(database: SupportSQLiteDatabase) {
        database.execSQL("ALTER TABLE web_apps ADD COLUMN new_field TEXT NOT NULL DEFAULT ''")
    }
}
// In DatabaseModule.kt:
Room.databaseBuilder(…)
    .addMigrations(MIGRATION_2_3)
    .build()
```

## Configuration

### KSP schema export

In `build.gradle.kts`:
```kotlin
ksp {
    arg("room.schemaLocation", "$projectDir/../app/schemas")
}
```

Room schema JSON files are committed to version control so that migration tests (`MigrationTestHelper`) can validate every upgrade path.

### SQLCipher passphrase

The passphrase is injected at `AppDatabase` construction time by `CryptoManager.databasePassphrase()`. No plaintext passphrase appears in source code or configuration files — it is generated once per device, encrypted with the Android Keystore, and stored in `SharedPreferences`.
